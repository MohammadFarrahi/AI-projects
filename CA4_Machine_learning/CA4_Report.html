These are used for feature scaling which is one of the most important data preprocessing step in machine learning. Algorithms that compute the distance between the features are biased towards numerically larger values if the data is not scaled. Normalization and Standardization are the most popular techniques for feature scaling. Also, feature scaling helps machine learning, and deep learning algorithms train and converge faster.

Normalization or Min-Max Scaling is used to transform features to be on a similar scale. The new point is calculated as :

$X_{new} = (X - X_{min})/(X_{max} - X_{min})$

Standardization or Z-Score Normalization is the transformation of features by subtracting from mean and dividing by standard deviation. This is often called as Z-score:

$X_{new} = (X - mean)/Std$

Since we have only _releaseyear as numerical feature and we don't have any understanding from disturbution of it, we use Normalization. because Normalizaion is normally useful when we dont know about the distribution. Although tree-based algorithms are fairly insensitive to the scale of the features.

There are two popular methods :

1 - Label Encoding : In this method each category is uniquely assigned to an integer. This technique is normally used when we have some kind of order for different values of categorical feature. For example, we can use this method to encode different levels of education.

2 - One-Hot Encoding : In this method, let's say we have 5 values for a categorical feature, we use a 5 bit number and for each value, a 5 bit number with all bits zero except 1 bit which is one will be assigned to it. This number is unique to this value of the feature. As you may notice, there is no order in this encoding and this method is useful when there is no order in values.

For Our data, Cause there is no order between values of categorical features, It seems, we need One-Hoe Encoding. But since there will be need for creating a great number of columns (each corresponds to each bit of n-bit encoded number), we use Label encoding. Also there will be no problem for our model scince we use desicion tree. Although, we should remember that there is no order between values.

df['genre2'] = df['genre2'].map(genre_types)
df['genre3'] = df['genre3'].map(genre_types)

In this part, in _listedin feature, a cell consists of at most five values, meaning each sample has at most five genres. So we decided to convert _listedin feature in to 3 (between 1 and 5) new features genre1, genre2 and genre3. Those samples that have more than 3 genres, the remain genres will be droped and those with less than 3 genres, we fill 2 or 1 other genres with value of former genre of sample. for exmaple if sample has only genre A, therefore : genre1 = genre2 = genre3 = A

In our model we used 80% of data for training 20% for testing. With this proportion we can have a good evaluation for our model; Because the portion of train data is big enough, so we don't face with problem of high bias and underfitting. Also poriton of test data is not that much low, so we can find out if our model has overfitting problem.

If we use very big portion of data like 98% for trainnig model, there is high chance that our model become train-data specific; It will act very well on train data with very high accuracy but not on other unseen data. it is called overfitting.

In our case, because the portion of test data is very very low and also because of high information gain of some features of data(like genres) this overfitting is not visible

If we use not big enough portion of data for tarining model (like 60%), our model won't be trained good enough and can't even have a good accuracy on train data. In fact, it will be difficault to find appropriate parametes and relationship between features and target in model. this is called underfitting

In our case, we can see we lower accuracy in comparison to when we had 80%-20% split on data.

Generally we expect RF to be a model with low variance in comparison to DT model. Because in RF we use more than one decision tree, and with different features and samples to decide about target.
